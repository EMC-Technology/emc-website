## 五、开源边界定义

### 5.1 边界判定原则

开源边界判定按以下优先级流程进行：

1. **PRT-01 最高保护**：涉及具体错误码的内容 → 不开源
2. **PRT-02 最高保护**：涉及模块映射的内容 → 不开源
3. **PRT-03 最高保护**：涉及内部实现的内容 → 不开源
4. **PRT-04 最高保护**：涉及安全机制的内容 → 不开源
5. **PRT-05 高保护**：涉及外部集成的内容 → 不开源
6. **PRT-06 高保护**：涉及性能优化的内容 → 不开源
7. 通过以上判定 → 可开源 (OSC-01 ~ OSC-06)

### 5.2 典型场景判定

| 场景 | 判定 | 类别 |
|------|------|------|
| 错误分为四级严重程度 | 可开源 | OSC-02 |
| 错误码格式为 ERR-[SRC]-[MOD]-[SEQ]_[SEV][SCO] | 可开源 | OSC-03 |
| ERR-AIM-LM-002 表示 AI 模型调用超时 | 受保护 | PRT-01 |
| AI 模型错误来源映射到 LM 模块 | 受保护 | PRT-02 |
| 使用 Arc<ErrorObject> 进行共享 | 受保护 | PRT-03 |
| 安全错误走独立审计通道 | 受保护 | PRT-04 |
| reqwest::Error 映射到 ERR-NET-* | 受保护 | PRT-05 |
| 使用指数退避策略控制重试延迟 | 可开源 | OSC-06 |
| jitter_factor=0.1 是最优参数 | 受保护 | PRT-06 |
